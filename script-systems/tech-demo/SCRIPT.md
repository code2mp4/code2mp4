# Tech Demo — Script System
> Category: Developer & Engineering
> Duration: 30-60s
> Scenes: 5-6
> Summary: Hook(title slam,0-5s)→Context(problem/value,5-15s)→Features(3items×5s,15-35s)→Proof(metrics,35-50s)→CTA(command+URL,50-60s). MAX 1 concept/scene, 15char headlines, 2 text elements visible at once. Voice: technical,precise,no marketing fluff. Numbers > adjectives. Anti-slop: no invented metrics, hook in first 1s, energy varies across arc.

## Narrative Arc
Hook → Context → Features → Proof → CTA

## Information Density
- ONE concept per scene (no mixing)
- Headlines: 8-15 characters max
- Subheads: 20-30 characters max
- Maximum 2 text elements visible at once
- Numbers > adjectives ("6 agents" beats "many agents")

## Pacing Map
| Section | Time | Duration | Feel |
|---------|------|----------|------|
| Hook | 0-5s | 5s | Fast slam — one bold statement |
| Context | 5-15s | 10s | Medium — problem or value |
| Features | 15-35s | 20s | Staggered — 3 items, 0.3s stagger |
| Proof | 35-50s | 15s | Slow — numbers count up |
| CTA | 50-60s | 10s | Confident — one decisive moment |

## Voice & Tone
- Technical, precise, confident
- No marketing fluff ("amazing", "revolutionary", "game-changing")
- Numbers and facts over claims
- Short declarative sentences
- Command-like CTA: "Build your first video" not "Why not give it a try?"

## Hook Patterns
1. **Title Slam** — "OPEN VIDEO" in large display font, single word or short phrase
2. **Question Hook** — "What if AI could render your video?" — answer comes in Context
3. **Stat Lead** — "6 agents. 5 motion systems. 1 command." — numbers first
4. **Visual First** — Strong abstract visual before any text appears (1s of silence)

## Scene Templates
### 1. Hook (0-5s)
- ONE bold statement or question
- Brand/product name visible within 0.5s
- No explanation, no subtext — just impact
- Glow, scale, or glitch flourish for energy

### 2. Context (5-15s)
- Problem statement OR value proposition
- "Describe a video. An AI agent builds it."
- One supporting line max
- Visual: abstract geometric, pipeline diagram, or screenshot

### 3. Features (15-35s)
- Option A (rapid): 3 features × 5s each, quick cuts
- Option B (deep): 1 feature × 20s, detailed
- Each feature: name + benefit (1 line each)
- Visual: matching icon, badge, or code block per feature

### 4. Proof (35-50s)
- Metrics if available: "49 tests" "4 agents detected" "CI passing"
- OR demonstration: "Watch it build"
- Maximum 3 numbers on screen at once
- Source attribution for any claims

### 5. CTA (50-60s)
- ONE call to action
- Command form: "Star us on GitHub" "Build your first video"
- URL or command visible: `github.com/openvideo-ai/openvideo`
- Logo lockup + license badge

## Anti-Slop Checklist
- [ ] No more than 2 text elements visible at once
- [ ] All numbers have context (what they measure)
- [ ] No invented metrics ("10× faster" without proof)
- [ ] No marketing adjectives
- [ ] Hook present in first 1s
- [ ] CTA is a command, not a suggestion
- [ ] Scene count matches arc (not all the same length)
- [ ] Energy varies across the arc (not flat)
